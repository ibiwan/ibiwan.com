Splitting Your Program into Separate Files

When writing programs, splitting the code into separate files makes it more readable and easier to organize.

The most common way to split a program with one class is into 3 files:

  1. A main program file
  2. A class header file
  3. A file containing the source code of your class functions

In the main class program file (often named main.cc), you should have your main functions:
//main.cc

#include(s) . . .
#include "classname.h"

void main()
{

    // . . . your code . . .

}
main.cc must include the class header file name within double quotations because it is located in the directory with main.cc


In the class header file (classname.h) the class and its members are declared.
//classname.h

#include(s) . . .


class class_name
{

private:
   int private_variable_1();
   char private_variable_2();

   void private_function()


public:
   int public_variable;

   void public_function_1();
   int public_function_2();

};


Thirdly, the source code of the class functions are located here (classname.cc).
//classname.cc

#include(s) . . .
#include "classname.h"

void class_name::private_function()
{

    . . .
    . . .
    . . .

}

void class_name::public_function_1()
{

    . . .
    . . .
    . . .

}

   // . . . etc . . .


Now that the code is split intro three files; how is compilation supposed to be done?

Makefiles!

This is a fourth file that tells the compiler which files to compile with what. Two examples are below:

A Simple MakeFile

An example of a makefile (named makefile) which has the following 3 files:
  1. main.cc
  2. classname.cc
  3. classname.h
The executable will be called main.
main: main.o classname.o
    g++ -g -o main main.o classname.o

main.o: main.cc classname.h
    g++ -g -Wall -c main.cc

classname.o: classname.cc classname.h
    g++ -g -Wall -c classname.cc



Generalized MakeFile

A different example of a makefile (named makefile) which has the following 3 files:
  1. main.cc
  2. list.cc
  3. list.h
The executable will be called mylist.
OBJS = list.o main.o
CC = g++    # C++ Compiler
CFLAGS = -g -Wall    # -g : in order to use debug

mylist: $(OBJS)
    $(CC) $(CFLAGS) -o mylist $(OBJS)

list.o: list.h list.cc
    $(CC) $(CFLAGS) -c list.cc

main.o: list.h main.cc
    $(CC) $(CFLAGS) -c main.cc

clean:
    rm -f mylist *.o core


To compile, type "make".

IMPORTANT NOTE: Each indented line must be indented with a TAB.
